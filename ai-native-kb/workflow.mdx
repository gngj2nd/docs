# Рабочий процесс с ИИ-агентом

Этот процесс описывает, как автор и ИИ-агент совместно работают над созданием документации в рамках AI-Native Knowledge Base.

### Шаг 1: Создание первичного описания

**Автор** создаёт первичный MDX-файл. В этом файле он в свободной форме описывает задачу, основные концепции и желаемую структуру будущей документации.

*Пример:*
```markdown
# Система аутентификации

## Проблема
Нужно описать, как работает наша новая система аутентификации на основе JWT.

## Структура
- Обзорный документ
- Как получить токен (эндпоинт /login)
- Как обновлять токен (эндпоинт /refresh)
- Описание структуры JWT-токена
```

### Шаг 2: Анализ и генерация документов ИИ-агентом

**ИИ-агент** получает в качестве контекста этот первичный файл. Он анализирует запрос и создаёт на его основе набор атомарных MDX-документов, придерживаясь заданной структуры. Агент может также использовать свои знания о проекте (доступ к коду) для обогащения контента.

*Результат работы агента:*
- `auth-system/index.mdx` (обзор)
- `auth-system/how-to-get-token.mdx`
- `auth-system/how-to-refresh-token.mdx`
- `auth-system/jwt-structure.mdx`

### Шаг 3: Ревью и редактирование

**Автор** проверяет сгенерированные документы. На этом этапе он может:
- Вносить правки в текст.
- Добавлять диаграммы (например, Mermaid.js).
- Уточнять детали, которые ИИ мог упустить.

### Шаг 4: Внесение изменений в базу знаний

После завершения редактирования **авт��р** коммитит изменения в Git-репозиторий. Это запускает CI/CD пайплайн, который может включать:
- Линтинг и проверку форматирования.
- Публикацию документации на портале (например, Mintlify).
- Индексацию новых документов для ИИ-агентов.

Этот итеративный процесс позволяет быстро создавать и поддерживать качественную, актуальную и легкодоступную базу знаний.
