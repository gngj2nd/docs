# Ключевые фичи

AI-Native Knowledge Base строится на наборе современных технологий и подходов, которые обеспечивают гибкость, версионируемость и интеграцию с ИИ.

### 1. Формат MDX

Использование MDX (Markdown + JSX) в качестве основного формата для документов даёт несколько преимуществ:
- **Гибкость:** Возможность вставлять интерактивные компоненты (диаграммы, API-референсы) прямо в документацию.
- **Независимость:** Легко перенести базу знаний на другую платформу или инструмент.
- **Удобство редактирования:**
    - **Web-редактор:** Интуитивно понятные редакторы, такие как Mintlify.
    - **IDE:** Работа с документацией как с кодом в привычной среде разработки.
    - **ИИ-агент:** Программное создание и редактирование файлов.

### 2. Git для контроля версий

Вся документация хранится в Git-репозитории:
- **Ревью:** Возможность организовать процесс код-ревью для изменений в документации.
- **Версионирование:** Чёткий контроль версий и истории изменений.
- **Интеграция:** Подключение базы знаний к ИИ-агентам и другим инструментам разработки.

### 3. ИИ-агент (например, Gemini CLI)

ИИ-агент — ключевой элемент системы, который берёт на себя рутинные задачи:
- **Работа с контекстом:** Сбор релевантной информации из базы знаний и исходного кода.
- **Написание документации:** Создание и структурирование файлов, написание и редактирование текстов.
- **Консультация:** Ответы на вопросы по базе знаний в режиме реального времени.

### 4. Автоматическая актуализация

Интеграция с CI/CD позволяет автоматически обновлять документацию по итогам работы над кодом, например, после мержа pull-request.

### 5. Интеграция в процессы разработки

База знаний становится активным участником всего цикла разработки:
- **Написание кода:** Предоставление контекста для ИИ-агентов.
- **Код-ревью:** Автоматическая проверка соответствия кода документации.
- **Тестирование:** Генерация тест-кейсов и unit-тестов на основе спецификаций.
- **Отладка и поддержка:** Быстрый доступ к релевантной информации для решения проблем.
